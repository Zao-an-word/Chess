# 五子棋游戏（C++ EasyX 实现）
基于 C++ 和 EasyX 图形库开发的经典五子棋小游戏，支持双人对战、人机对战（AI 带评分算法）、胜利判定、难度等级、先手选择等核心功能，适配 Windows 平台（Visual Studio/Dev-C++）。

## 📋 项目介绍
### 核心功能
- 对战模式：支持双人对战、人机对战（AI 基于评分算法落子）
- 先手选择：人机模式可选择玩家/AI 先手（黑棋/白棋）
- 胜利判定：自动检测横/竖/斜向五子连线，绘制胜利红线并弹窗提示
- 难度等级：胜利后自动提升难度等级，可视化等级进度条
- 交互体验：鼠标悬浮预览落子位置、背景音效、开局菜单图片交互
- 评分算法：AI 基于落子位置评分（活四/活三/双活三等），优先攻防最优解
- 棋盘系统：标准 15×15 棋盘，天元/星位标记，字母+数字坐标标注

### 技术栈
- 编程语言：C++（C++11 及以上）
- 图形库：EasyX Graphics Library（Windows 专属）
- 音频支持：Windows Multimedia API（mmsystem.h）
- 开发环境：Visual Studio 2022 / Dev-C++（推荐 VS2022）

## 🛠️ 环境配置
### 前置条件
1. 操作系统：Windows 7/10/11（EasyX 仅支持 Windows）
2. 编译器：Visual Studio 2022（MSVC）/ Dev-C++（MinGW）
3. EasyX 图形库：
   - 访问 [EasyX 官网](https://easyx.cn) 下载对应编译器版本的安装包
   - 运行安装包，自动完成库文件配置（VS 需管理员权限）

### 快速配置（Visual Studio）
1. 打开 Visual Studio 2022，新建「空项目」；
2. 将核心代码文件（`gobang.cpp`）添加到项目中；
3. 项目属性设置：
   - 字符集：改为「使用 Unicode 字符集」；
   - 链接器：添加 `winmm.lib`（代码中已用 `#pragma comment(lib,"winmm.lib")` 自动链接）；
4. 点击「本地 Windows 调试器」编译运行。

### 快速配置（Dev-C++）
1. 打开 Dev-C++，新建「C++ 项目」；
2. 将核心代码粘贴到项目文件中，保存为 `gobang.cpp`；
3. 工具 → 编译选项 → 勾选「支持 C++11 标准」；
4. 直接点击「运行」按钮编译执行。

- 棋盘规格：15×15 格子，每个格子 40×40 像素

- AI 落子逻辑：基于位置评分+攻防优先级，优先堵玩家活四/活三，其次自身造杀
- 难度升级：每胜利一局等级 +1，等级进度条可视化展示

## 📂 代码结构
gobang.cpp
- ├── 全局常量/变量：棋盘尺寸、玩家/AI 标识、分数地图、胜利标记等核心变量
- ├── 核心函数：
- │ ├── tiyu()：绘制棋盘天元/星位（5个关键点）
- │ ├── block()：绘制棋盘网格线（横/竖线）
- │ ├── text()：绘制棋盘坐标标注（A-O 列，1-15 行）
- │ ├── showpiece()：绘制已落子的所有棋子（区分黑白）
- │ ├── level()：绘制难度等级进度条+等级图标
- │ ├── ver/cro/xie1/xie2()：检测横/竖/斜向五子连线，判定胜利
- │ ├── vic()：统一调用胜利检测函数，返回胜利方标识
- │ ├── bk()：绘制游戏背景（棋盘+天元+等级+坐标+棋子）
- │ ├── vicshow()：胜利弹窗提示，处理重开/退出逻辑
- │ ├── close()：计算鼠标位置最近的棋盘格点，实现精准落子
- │ ├── piece()：绘制单个棋子（黑白纯色圆）
- │ ├── clear()：重置游戏状态（清空棋盘、分数、胜利标记）
- │ ├── begin()：开局菜单（图片交互+背景音乐+模式选择）
- │ ├── setai()：人机模式先手选择弹窗
- │ ├── cal_unit()：AI 核心评分函数（计算单个位置的攻防分数）
- │ ├── computer()：AI 遍历棋盘，选择评分最高位置落子
- │ ├── pk()：双人对战主逻辑（鼠标落子+交替下棋+胜利检测）
- │ ├── aipk()：人机对战主逻辑（玩家落子+AI 响应+胜利检测）
- └── main()：程序入口（开局菜单 → 模式选择 → 对战逻辑）

## 🧠 AI 评分算法核心逻辑
### 评分优先级（从高到低）
1. **成五**：5 子连线，直接胜利（评分 5000000）
2. **双活四**：两个活四（两端空），必赢（评分 4000000）
3. **活四**：单个活四（两端空），高优先级（评分 800000）
4. **冲四**：有一端被堵的四连（评分 35000）
5. **双活三**：两个活三（两端空），高威胁（评分 400000）
6. **活三**：单个活三（两端空），造杀关键（评分 60000）
7. **双活二**：两个活二（两端空），布局核心（评分 20000）
8. **活二**：单个活二（两端空），基础布局（评分 10000）
9. **冲三**：有一端被堵的三连（评分 5000）

### 落子策略
- AI 先计算自身落子得分，再计算玩家落子得分；
- 最终评分 = AI 得分 - 玩家得分，选择评分最高位置落子；
- 优先防守（堵玩家高威胁落子），其次进攻（自身造杀）。

## 🐞 常见问题
### 1. 编译报错“找不到 graphics.h”
- 原因：未安装 EasyX 图形库，或安装版本与编译器不匹配；
- 解决：重新下载对应编译器版本的 EasyX，运行安装包修复。

### 2. 图片/音频无法加载
- 原因：`resources` 文件夹路径错误，或文件缺失/格式不支持；
- 解决：
  - 确认 `resources` 文件夹与可执行文件同级，文件名与代码中一致；
  - 音频仅支持 Windows 兼容格式（MP3/WAV/OGG），图片支持 BMP/JPG/PNG。

### 3. 鼠标落子位置不准确
- 原因：`close()` 函数格点计算逻辑未适配屏幕坐标；
- 解决：代码中 `close()` 已实现“最近格点”计算，无需额外修改，点击时尽量靠近格子中心。

### 4. AI 落子无响应
- 原因：棋盘未清空导致评分计算异常，或 `chess` 数组越界；
- 解决：`clear()` 函数已重置所有游戏状态，AI 落子前会先清空评分数组，确保计算有效。

### 5. 宽字符报错（如 MessageBoxW/_stprintf_s）
- 原因：项目字符集未设置为 Unicode；
- 解决：VS 项目属性 → 高级 → 字符集 → 改为「使用 Unicode 字符集」。
